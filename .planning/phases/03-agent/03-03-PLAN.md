### 两次 humanApproval 实现策略
由于 Agent 内部完成了 LLM + 图像生成 + sharp 切割的完整流程，两次 humanApproval 需要在 Node 层实现：

**方案**：在 Node 中分阶段调用 Agent
1. **第一阶段**：只调用 LLM 生成分镜 JSON（新增一个选项或拆分函数）
2. **第一次 humanApproval**：用户确认/编辑分镜 JSON
3. **第二阶段**：调用图像生成和切割
4. **第二次 humanApproval**：用户确认分镜图

**简化方案**（推荐）：保持 Agent 完整调用，在 Node 中通过状态标记实现两次检查点：
- 如果 `state.storyboardPlan` 存在但 `state.step4_video` 不存在 → 第一次 humanApproval（JSON 确认）
- 如果 `state.step4_video` 存在但用户要求重试图像 → 第二次 humanApproval（图像确认）

### Node 输入/输出
输入：`WorkflowState`，包含：
- `step2_characters`（艺术总监输出）
- `step3_storyboard`（选角导演输出，含 image_url）
- `step1_script`（剧本内容）
- `videoSpec`（视频规格）

输出：`Partial<WorkflowState>`，包含：
- `step4_video`（分镜结果）
- `currentStep: 5`
- `humanApproval`（导演模式时设置）

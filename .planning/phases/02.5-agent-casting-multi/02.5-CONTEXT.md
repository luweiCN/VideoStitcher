# Phase 2.5: 选角导演多阶段架构重构

## 目标

将当前单阶段的选角导演 Agent 拆分为多阶段架构（Planner + Visualizer），与分镜设计 Agent 架构保持一致。

## 背景

当前选角导演 Agent 是单阶段实现：
1. 接收艺术总监输出（角色设定）
2. 直接生成图像提示词并调用图像生成

这种设计无法灵活配置不同阶段使用不同模型，也不符合 GSD 多阶段架构标准。

## 决策记录

### D-01: 阶段划分

**决策**：拆分为两个阶段

- **阶段 1 - Casting Planner（选角规划器）**
  - Agent ID: `casting-director-planner`
  - 模型类型: LLM
  - 职责:
    - 分析艺术总监输出的角色设定
    - 为每个角色生成详细的视觉描述（外貌、服装、姿势）
    - 确定角色之间的视觉一致性规则
    - 输出结构化的角色视觉规格

- **阶段 2 - Casting Visualizer（形象可视化器）**
  - Agent ID: `casting-director-visualizer`
  - 模型类型: image_generation
  - 职责:
    - 接收 Planner 的角色视觉规格
    - 生成角色参考图（Character Reference Sheet）的图像生成提示词
    - 调用图像生成 API
    - 返回角色参考图

### D-02: 提示词分层

遵循三层架构：
- **EDITABLE**: Agent 人设、方法论、风格规则
- **LOCKED**: JSON 输出格式、图像提示词模板、自检清单
- **DYNAMIC**: 运行时变量（角色设定、视觉规格等）

### D-03: 与现有代码的关系

- 保留现有的 `casting-director` 单阶段实现作为兼容版
- 新增 `casting-director-planner` 和 `casting-director-visualizer` 两个阶段
- 在 BUILTIN_PROMPT_TEMPLATES 中注册新阶段
- LangGraph Node 可选择使用多阶段或单阶段版本

## 范围

### In Scope
- 创建 castingDirectorMultiTemplates.ts 三层提示词常量文件
- 实现 casting-director-planner 和 casting-director-visualizer Agent
- 注册 BUILTIN_PROMPT_TEMPLATES
- 重构 LangGraph Node 支持多阶段调用

### Out of Scope
- 删除现有单阶段实现（保留兼容）
- 修改艺术总监 Agent
- 修改 PromptStudio 核心逻辑

## 成功标准

1. Planner 能正确分析角色设定并生成视觉规格
2. Visualizer 能根据规格生成高质量角色参考图
3. PromptStudio 能分别配置两个阶段
4. 两个阶段可使用不同模型（Planner 用 LLM，Visualizer 用图像生成模型）
5. 导演模式支持 `humanApproval` 暂停

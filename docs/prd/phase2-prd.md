## 1. 产品概述

### 1.1 产品定位

**AI 驱动的营销视频批量生产工具**

VideoStitcher 第二阶段将从"视频剪辑工具箱"升级为"智能视频生产平台"，通过 LangGraph 状态机自动编排多个 AI Agent，实现从脚本生成到视频导出的完整批量生产流程。

### 1.2 目标用户

- **主要用户**: 短视频营销从业者、电商运营、内容创作者
- **次要用户**: 中小企业营销团队、广告代理商
- **用户痛点**:
  - 脚本创作耗时长、质量不稳定
  - 视频制作门槛高、成本高
  - 批量生产效率低
  - 难以复用高转化率素材

### 1.3 核心价值主张

1. **批量生产**: 一次配置，批量生成多条脚本和视频
2. **AI 辅助**: 从脚本到视频全流程 AI 驱动
3. **知识积累**: 自动学习高转化素材，持续优化输出
4. **低门槛**: 无需视频制作经验，对话式操作

## 2. 核心功能模块

### 2.1 脚本批量生成 (Script Generator)

#### 2.1.1 功能描述

通过 AI 批量生成营销视频脚本，支持多种风格和参数配置。

#### 2.1.2 详细需求

**输入参数**:
- 风格选择: 幽默、悬疑、搞笑、教学、解说（5 种预设）
- 地区配置: 中国大陆、香港、台湾、东南亚等（影响语言和文化习惯）
- 产品名称: 文本输入
- 生成数量: 3/5/10 条（可选）
- 自定义要求: 可选文本输入框

**输出结果**:
- 脚本列表（包含 ID、文本、风格标签）
- 每条脚本支持:
  - 查看详情
  - 编辑修改
  - 重新生成
  - 存入待产库

**交互流程**:
1. 用户选择风格
2. 填写产品名称和地区
3. 选择生成数量
4. 点击"生成脚本"
5. 显示加载动画（预计 5-15 秒）
6. 展示生成的脚本列表
7. 用户可编辑/重新生成单条
8. 确认后存入待产库

**技术实现**:
- 调用 LangGraph `ScriptNode`
- 使用豆包 LLM API 生成文案
- RAG 检索知识库相关素材（如果有）
- 返回结构化 JSON 数据

**验收标准**:
- ✅ 支持至少 5 种风格
- ✅ 生成速度 < 15 秒（3 条）
- ✅ 每条脚本字数 50-200 字
- ✅ 支持单条重新生成（不影响其他条）
- ✅ 脚本内容不重复率 > 80%

### 2.2 导演模式 (Director Mode)

#### 2.2.1 功能描述

精细化视频创作流程，从脚本到分镜到成片，支持用户深度参与和调整。

#### 2.2.2 详细需求

**输入参数**:
- 脚本 ID（从待产库选择）
- 视频规格:
  - 时长: 15s / 30s / 60s
  - 比例: 16:9（横版） / 9:16（竖版） / 1:1（方形）

**输出结果**:

**阶段 1: 人物设定**
- 人物数量: 根据脚本自动生成 1-3 个角色
- 每个角色包含:
  - 角色名称
  - 外貌描述
  - 性格特征
  - 概念图（AI 生成）

**阶段 2: 分镜矩阵**
- 分镜数量: 根据时长自动拆分（15s=3-5 镜，30s=6-10 镜，60s=12-20 镜）
- 每个分镜包含:
  - 场景编号
  - 场景描述
  - 人物动作
  - 镜头语言
  - 概念图（AI 生成）

**阶段 3: 视频渲染**
- 视频预览: 低分辨率预览（480p）
- 最终导出: 高分辨率（1080p / 720p 可选）
- 渲染进度: 实时显示百分比

**交互流程**:
1. 用户从待产库选择一条脚本
2. 配置视频规格（时长、比例）
3. 点击"开始创作"
4. **人物设定阶段**（预计 20-40 秒）:
   - 显示加载进度
   - 展示人物设定卡片
   - 用户可编辑人物特征
   - 可重新生成概念图
   - 确认后进入下一阶段
5. **分镜阶段**（预计 1-3 分钟）:
   - 分屏布局：左侧对话 + 右侧画布
   - 画布显示分镜节点（可拖拽、缩放）
   - 每个节点显示概念图 + 描述
   - 用户可调整分镜顺序
   - 可重新生成单个分镜图
   - 确认后进入视频渲染
6. **视频渲染阶段**（预计 5-10 分钟，异步）:
   - 显示任务队列
   - 实时进度条
   - 支持后台运行
   - 完成后通知用户
7. 导出视频到本地

**技术实现**:
- 调用 LangGraph 状态机:
  - `CharacterNode` → 豆包 Vision API
  - `StoryboardNode` → 豆包 Vision API
  - `VideoNode` → 火山视频 API
- 任务队列管理（支持多任务并行）
- WebSocket 实时推送进度

**验收标准**:
- ✅ 人物设定生成速度 < 40 秒
- ✅ 分镜生成速度 < 3 分钟（10 镜）
- ✅ 视频渲染支持后台运行
- ✅ 画布支持拖拽、缩放、连线
- ✅ 概念图分辨率 ≥ 512x512
- ✅ 最终视频分辨率 ≥ 1080p

### 2.3 知识库上传 (Knowledge Base)

#### 2.3.1 功能描述

上传历史高转化视频和文案，AI 自动分析素材特征，应用到新脚本生成。

#### 2.3.2 详细需求

**输入**:
- 视频文件: MP4 / MOV（最大 500MB）
- 文案文件: TXT / DOCX
- 标签: 可选（如"高转化"、"护肤品"、"促销"等）

**处理流程**:
1. 文件上传到本地
2. 视频提取关键帧（每 2 秒 1 帧）
3. 文案向量化（DoubaoEmbeddings）
4. 存储到 SQLite + sqlite-vec
5. 生成素材卡片

**输出**:
- 素材列表（包含 ID、缩略图、文案摘要、标签）
- 向量化状态
- 相似度检索结果

**交互流程**:
1. 拖拽或点击上传视频 + 文案
2. 显示上传进度
3. 显示 AI 分析进度（预计 1-3 分钟/条）
4. 展示素材卡片
5. 用户可添加/编辑标签
6. 素材自动应用到脚本生成（RAG 检索）

**技术实现**:
- 文件上传: Electron 文件系统 API
- 向量化: DoubaoEmbeddings
- 存储: SQLite + sqlite-vec
- 检索: 余弦相似度搜索

**验收标准**:
- ✅ 支持批量上传（最多 10 条）
- ✅ 向量化速度 < 3 分钟/条
- ✅ 检索响应速度 < 2 秒
- ✅ 向量维度 ≥ 1024
- ✅ 支持标签管理

## 5. 数据模型

### 5.1 脚本 (Script)

```typescript
interface Script {
  id: string;                    // UUID
  text: string;                  // 脚本文本
  style: '幽默' | '悬疑' | '搞笑' | '教学' | '解说';
  productId?: string;            // 关联产品
  region: string;                // 地区
  createdAt: Date;
  updatedAt: Date;
  status: 'draft' | 'pending' | 'producing' | 'completed';
}
```

### 5.2 项目 (Project)

```typescript
interface Project {
  id: string;
  scriptId: string;
  videoConfig: {
    length: 15 | 30 | 60;
    ratio: '16:9' | '9:16' | '1:1';
  };
  characters: Character[];
  storyboard: StoryboardItem[];
  videoUrl?: string;
  status: 'draft' | 'character' | 'storyboard' | 'rendering' | 'completed';
  createdAt: Date;
  updatedAt: Date;
}
```

### 5.3 知识库素材 (KnowledgeItem)

```typescript
interface KnowledgeItem {
  id: string;
  videoPath: string;
  textContent: string;
  embedding: number[];           // 向量
  tags: string[];
  createdAt: Date;
}
```

## 6. 技术架构

### 6.1 前端技术栈

- **框架**: React 18 + TypeScript
- **构建工具**: Vite 5
- **样式**: Tailwind CSS 3.4
- **状态管理**: Zustand
- **UI 组件**: 自定义组件库
- **画布**: React Flow（导演模式）

### 6.2 后端技术栈

- **运行时**: Node.js（Electron 主进程）
- **AI 编排**: LangGraph.js + LangChain
- **向量数据库**: SQLite + sqlite-vec
- **AI API**: 火山引擎（豆包 LLM + Vision + Video）

### 6.3 通信架构

- **前端 ↔ 主进程**: Electron IPC
- **主进程 ↔ LangGraph**: 直接调用
- **LangGraph ↔ AI API**: HTTP/REST

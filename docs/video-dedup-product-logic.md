# 视频降重处理产品逻辑

本文档保存“视频降重处理”当前已经确认的产品规则。处理引擎与任务中心已经接入；后续实现与验收应以本文档和根目录 `CONTEXT.md` 中的统一术语为准。

正式领域术语继续使用“叠加素材、素材事件、素材分布”等现有名称，不修改内部产品模型；用户界面使用“变体元素/元素”作为展示文案，避免与待处理的原视频素材混淆。

## 1. 产品定位

- 入口名称：`视频降重处理`
- 入口说明：`批量生成视频变体，用于降低素材重复度`
- 核心目标：在不让画面显得杂乱的前提下，通过改变叠加素材、出现时间和画面位置，批量生成不同的视频变体。
- 当前阶段：UI、元素库、约束排程、绿幕配方和 FFmpeg 处理引擎均已接通，任务通过任务中心执行。

## 2. 首版素材范围

- 静态贴纸：PNG、WebP、JPG、JPEG、BMP、AVIF。
- 动态贴纸：GIF。
- 绿幕视频：MP4，后续可兼容 MOV。
- 素材统一来自用户指定的本地素材库根目录。
- 设计师不需要预先导出体积巨大的透明底视频；绿幕素材保留压缩视频格式，由处理引擎在合成时按抠色配方处理。

## 3. 视频变体生成

- `每条生成`：每条原视频需要输出的视频变体数量。
- `出现次数`：每条视频变体需要安排的素材事件总数。
- 每个素材事件只能选择一个叠加素材。
- 每个视频变体应重新随机素材、出现时间和画面位置，从而形成差异。
- UI 使用“出现次数”，不使用“素材出现次数”或“叠加素材条数”。

## 4. 素材事件持续时间

- `最短持续`：单个素材事件允许的最短持续秒数。
- `最长持续`：单个素材事件允许的最长持续秒数。
- 每个素材事件的实际持续时间在最短值和最长值之间随机产生。
- 首版默认每条视频出现 3 次，每次持续 4～5 秒。
- 生成规则首次使用采用产品默认值；用户修改“每条生成、出现次数、最短持续、最长持续”后，本地保存最后一次设置，并在再次打开软件时恢复。
- UI 不再使用“最短出现”和“最长出现”，避免与开始出现的时间点混淆。

## 5. 素材分布

### 5.1 固定规则：单轨防重叠

- 单轨防重叠固定开启，不提供关闭选项。
- 任意时间点最多只能显示一个叠加素材。
- 一个素材事件尚未结束时，不得开始另一个素材事件。
- 前后两个素材事件之间必须满足`素材最小间隔`。
- 默认素材最小间隔为 1 秒。
- 该规则优先级高于随机性；不能为了满足出现次数而强行重叠素材。

### 5.2 有效时间范围

- `跳过片头`：视频开头禁止安排素材事件的时间长度，默认 1 秒。
- `跳过片尾`：视频结尾禁止安排素材事件的时间长度，默认 1 秒。
- 有效时间范围为：`跳过片头`之后，到`视频总时长 - 跳过片尾`之前。
- 所有素材事件必须完整落在有效时间范围内。

### 5.3 均匀分布

- 均匀分布是默认且推荐的分布方式。
- 根据素材出现次数，将有效时间范围划分为相应数量的分布时段。
- 每个时段内部随机选择一个可容纳完整素材事件的开始时间。
- 所有素材事件仍须满足单轨防重叠和素材最小间隔。
- 目标是让素材自然分散在视频前、中、后段，避免素材过度集中。

### 5.4 随机分布

- 在整个有效时间范围内随机选择素材事件的开始时间。
- 随机结果必须满足单轨防重叠、素材最小间隔和有效时间范围。
- 出现冲突时应重新抽取时间，不得保留重叠结果。
- 随机分布用于提高不同视频变体之间的时间差异，但不保证前、中、后段均匀出现。

### 5.5 手动调整预览时间轴

- 真实预览生成后，用户可以拖动整个元素事件块，调整它在视频中的出现位置。
- 用户可以拖动事件块左右两端，调整开始时间、结束时间和持续时长。
- `最短持续`和`最长持续`只约束自动排程，不限制用户手动拉伸事件时长。
- 手动调整的事件最低持续 0.1 秒，并继续遵守跳过片头、跳过片尾、元素最小间隔和单轨防重叠约束，不允许拖出有效范围或与相邻事件重叠。
- 时间轴发生修改后，现有视频标记为待更新，主按钮改为`重新渲染`。
- 点击`重新渲染`时必须使用用户调整后的完整事件表，不得重新随机素材、位置或时间。
- 已重新渲染的手动时间轴在加入任务时应用于首条原视频的第 1 个变体；同批次其他变体继续按当前规则随机生成，以保留批量差异性。
- 静态贴纸、GIF 动态贴纸、绿幕视频在图例和事件块中分别固定使用蓝色、紫色、绿色，不跟随全局主题色合并。
- `增加变体`从完整元素库随机抽取一个元素，并优先放入可容纳 3 秒事件的随机空档；空档不足 3 秒时缩短为当前可容纳时长。
- 新增事件继续遵守片头片尾、元素最小间隔和单轨防重叠约束；没有任何空档时提示用户先移动或缩短现有事件。
- 未选中的事件保持蓝色、紫色或绿色分类色；点击事件后，该事件切换为红色选中态，同时启用红色的`删除所选`按钮。
- 为保证视频降重任务有效，时间轴至少保留一个事件；需要替换最后一个事件时，应先增加新事件再删除旧事件。

## 6. 画面位置

- 素材分布只决定素材什么时候出现。
- 画面位置决定素材显示在视频画面的哪里，两者是独立规则。
- 首版位置范围为左上、右上、左下和右下四个安全区域。
- 每个素材事件可在已启用的位置范围中随机选择一个位置。
- 后续可扩展自由坐标、逐元素随机缩放、边距和画面避让区域。

## 7. 本地素材库

- 用户选择一个固定的本地素材库根目录。
- 推荐按静态贴纸、GIF 动态贴纸和绿幕视频分类存放。
- 文件夹只用于人工整理，不参与元素类型判断。
- 元素类型只按扩展名判断：GIF 归入 GIF 动态贴纸，MP4/MOV/M4V 归入绿幕视频，其余受支持图片归入静态贴纸。
- “全部”视图按静态贴纸、GIF 动态贴纸和绿幕视频分组展示，不使用无分类的平铺列表。
- 软件递归扫描元素库、按类型统计内容、生成列表缩略图，并统计可用元素、缺少配方和异常文件。
- 每个绿幕视频可以保存独立的绿幕抠色配方。
- 绿幕抠色配方至少包含基准色、相似度、边缘柔化和绿色溢出抑制。

## 8. 已确认的 UI 展示文案

正式领域名称保持不变；以下替换只作用于用户界面和面向用户的说明文案。

| 原文案 | 确认文案 |
|---|---|
| 时间编排 / 素材分布 | 元素分布 |
| 等分时间槽 | 均匀分布 |
| 全局随机 | 随机分布 |
| 素材库 | 变体元素库 |
| 素材库状态 | 元素库状态 |
| 素材出现 | 出现次数 |
| 素材最小间隔 | 元素最小间隔 |
| 最短出现 | 最短持续 |
| 最长出现 | 最长持续 |

元素分布模块说明固定为：`控制变体元素在视频时间线中的出现位置与间隔。`

单轨规则提示固定为：`单轨防重叠已固定开启，任意时间只出现一个变体元素。`

## 9. 当前引擎默认行为

以下是为保证首版可完整运行而采用的实现默认值，后续可以继续产品化配置：

- 当有效时间不足以容纳设定的出现次数、最短持续时间和最小间隔时，阻止创建任务并提示调整参数，不自动削减出现次数。
- GIF 或绿幕视频短于元素事件持续时间时循环播放，直到覆盖完整事件时长。
- 同一条视频变体允许重复抽取同一个元素。
- 每个任务保存随机种子和完整元素事件排程，失败重试可以复现同一结果。
- 变体元素按原视频宽度等比缩放，默认值为 22%。
- 首版提供小（12%）、中（22%）、大（32%）三个快捷尺寸，并支持 5%～50% 自定义尺寸。
- 尺寸是当前批次的全局参数，同一批任务中的所有变体元素使用相同尺寸。
- 效果预览与正式导出必须使用同一个尺寸参数；尺寸变化后，已有预览标记为需要重新生成。
- 首版不改变元素透明度，不添加进出场动画。

## 10. 后续可选增强

- 将逐元素缩放比例、透明度和进出场动画开放为高级参数。
- 增加“不重复抽取同一元素”的可选策略。
- 增加自由坐标、画面主体识别和避让区域。
